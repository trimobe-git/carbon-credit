## 1. Criação da Publicação

### Passo a passo

1. Acesse [substack.com](https://substack.com) e clique em **"Start writing"**
2. Crie uma conta com o email `contato@trimobe.com`
3. Na tela de configuração inicial, preencha os campos conforme a tabela abaixo
4. Escolha o tema visual (será customizado na seção 3)
5. Pule a etapa de importação de contatos (faremos manualmente depois)
6. Pule a etapa de convidar escritores (publicação individual)

### Campos de cadastro

| Campo Substack | Valor a preencher |
|---|---|
| Publication name | **Carbono no Agro** |
| Custom URL | `carbononoagro.substack.com` |
| Email de login | contato@trimobe.com |
| Idioma da publicação | Português |
| Tipo de publicação | Newsletter |

> **Por que "Carbono no Agro" e não "TRIMOBE"?** O nome da publicação deve comunicar o conteúdo, não a empresa. "Carbono no Agro" é descritivo, facilita a descoberta na rede do Substack, e posiciona a newsletter como referência do setor. A marca TRIMOBE aparece no byline e na página About.

## 13. Domínio Customizado

O Substack permite usar um domínio customizado em vez de `carbononoagro.substack.com`.

### Configuração recomendada

| Configuração | Valor |
|---|---|
| Domínio customizado | `newsletter.trimobe.com` |
| Tipo de registro DNS | CNAME |

### Passo a passo

1. No Substack, acesse **Settings > Publication details > Custom domain**
2. Insira `newsletter.trimobe.com`
3. O Substack fornecerá um valor de CNAME (geralmente `target.substack-custom-domains.com`)
4. No painel DNS do domínio trimobe.com (ex: Vercel, Cloudflare):
   - Adicione um registro CNAME:
     - **Host:** `newsletter`
     - **Value:** o valor fornecido pelo Substack
     - **TTL:** Auto
5. Aguarde propagação DNS (pode levar até 48 horas)
6. Volte ao Substack e clique em **Verify**

> **Nota:** O domínio customizado no Substack é gratuito — não requer plano pago. Após configurar, os emails serão enviados de `newsletter.trimobe.com` e a URL da publicação será `newsletter.trimobe.com`.

## 16. Notas sobre Integração com Automação

### Mudança de Resend para Substack

O plano original previa Resend + React Email para envio de newsletters. Com a migração para Substack, o pipeline de automação se adapta:

| Etapa | Plano original (Resend) | Plano atual (Substack) |
|---|---|---|
| Curadoria de notícias | Google Alerts → n8n → Claude → Notion | Sem mudança |
| Geração de draft | Claude API → Notion | Sem mudança |
| Formatação | React Email template (automático) | Fundador cola no editor do Substack |
| Envio | Resend API (automático via n8n) | Publicação manual no Substack |
| Analytics | Resend dashboard | Substack dashboard (nativo) |
| Gestão de assinantes | Lista manual / Notion | Substack (nativo, com importação/exportação) |
| Descoberta/crescimento | Apenas via LinkedIn e site | Rede do Substack + LinkedIn + site |

### Fluxo semanal adaptado

```
[Dom-Seg] Google Alerts coletam notícias → n8n → Notion
                                                ↓
[Segunda 8h] Claude classifica relevância (1-10), gera resumos
                                                ↓
[Segunda 9h] Claude gera draft completo da newsletter + 3 posts LinkedIn
                                                ↓
                                        [Notion: status "Aguardando revisão"]
                                                ↓
[Terça manhã] Fundador revisa draft no Notion
                                                ↓
              Fundador copia texto para o editor do Substack
                                                ↓
              Fundador publica no Substack (10h)
                                                ↓
[Terça-Sexta] Fundador agenda posts LinkedIn no Buffer
```

### Publicação via email (alternativa futura)

O Substack permite publicar enviando um email para um endereço específico da publicação. Isso abre a possibilidade de automação futura:

1. Claude gera draft → Notion
2. Fundador aprova no Notion (muda status)
3. n8n detecta a mudança de status → envia o conteúdo para o email de publicação do Substack
4. Newsletter é publicada automaticamente

> **Por ora, manter o fluxo manual:** copiar do Notion para o editor do Substack. Automatizar apenas quando o processo estiver rodando há pelo menos 4-6 semanas e o formato estiver estabilizado.

### Substack API

O Substack não oferece API pública oficial para publicação programática. As opções de automação são:
- **Email-to-publish** (descrito acima)
- **Zapier/IFTTT** (integrações limitadas)
- **Export de assinantes** (CSV, para sincronizar com Notion CRM)

Para a fase atual do negócio, o fluxo manual (Claude gera → Notion armazena → fundador publica no Substack) é suficiente e mantém o princípio human-in-the-loop.
